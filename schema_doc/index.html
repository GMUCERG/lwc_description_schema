Design

Captures attributes of an LWC hardware implementations to enable automated benchmarking and validation.

name: string , must match ^[A-Za-z][-A-Za-z_0-9]+$

A unique identifier for the design. It can consist of English letters, digits, dashes, and underscores and must start with a letter.

description: string

A short description of the design.

authors: string[]

A list of names of author(s) for this implementation.

string
url: string (uri)

Uniform Resource Locator pointing to a webpage or source repository associated with this design or its author(s).

rtl: object

Details of the synthesizable RTL design

sources: string[]

Non-empty list of HDL source file paths in correct compilation order. All paths must be relative to the location of the configuration file. Path separator is '/' (slash) on all platforms. Paths are case-sensitive and should not contain whitespaces. HDL language is inferred from the file extension. NOTE: Xeda sources fields can be an object containing meta-data such as HDL type and version.

string
includes: string[]

Non-empty list of HDL include file paths, such as Verilog headers. Include files are not directly compiled but need to be present for elaboration of design. Order is arbitrary. All paths must be relative to the location of the configuration file. Path separator is '/' (slash) on all platforms. Paths are case-sensitive and should not contain whitespaces.

string
top: string LWC

Name of top-level RTL entity/module.

clock: object [object Object]

Top level RTL clock signal. Only a single clock is supported by LWC API

port: string clk

Name of the top-level RTL clock input.

reset: object [object Object]

Top level reset signal. Only a single reset is supported by LWC API.

port: string reset

Name of top-level RTL reset input.

active_high: boolean true

Polarity of the reset signal. Active-high (positive) if true, otherwise active-low.

synchronous: boolean true

Reset is synchronous (positive) if true, otherwise asynchronous.

parameters: object

Top-level design parameters or generics specified as a key-value map. The default value of each parameter is overridden by synthesis tool, simulator, testbench, or wrapper. For the best tool compatibility, we only support integer and string values.

language: object

Information about Hardware Description/Design Language(s) used.

vhdl: #/$defs/vhdl
verilog: #/$defs/verilog
systemverilog: #/$defs/systemverilog
tb: object

Details of test-bench used for verification of top-level design. [Optional]

sources: string[]

Source files used only for verification. Should NOT contain any of the files included in 'rtl.sources'.

string
includes: string[]

HDL include file paths

string
top: string

Name of top-level test entity or module.

parameters: object

Testbench parameter or generics specified as a key-value map. The default value of each parameter is overridden by the simulator. For the best tool compatibility, we only support integer and string values.

language: object

Information about HDL or programming languages used in the testbench.

vhdl: #/$defs/vhdl
verilog: #/$defs/verilog
systemverilog: #/$defs/systemverilog

lwc: object

LWC-specific meta-data

algorithm: string,array

LWC AEAD/Hash algorithm(s) supported by this design. Should follow SUPERCOP naming conventions and uniquely identify the scheme's variant and version. Example: 'romulusn1v12'

pdi_order: object [object Object]

Order in which different input segment types should be fed to PDI.

enc: object[] npub,ad,data,tag
object , x ∈ { ad , data , npub , tag }
dec: object[] npub,ad,data,tag
object , x ∈ { ad , data , npub , tag }
pt_block_bits: integer , { x | x 1 }

Algorithm's size of plaintext/ciphertext 'blocks' in bits. This is the number of bits that the algorithm operates upon during its basic operations.

ad_block_bits: integer , { x | x 1 }

Algorithm's size of associated-data 'blocks' in bits. This is the number of bits that the algorithm operates upon during its basic operations.

key_bits: integer 128
npub_bits: integer 128
tag_bits: integer 128
hash_bits: integer 128
ports: object

Description of LWC ports.

pdi: object

Public Data Input port

bit_width: integer , { x | 8 x 32 }

Width of each share of PDI data (bits). Width of 'pdi_data' port is 'pdi.bit_width * pdi.shares'.

num_shares: integer , { x | 1 x 8 } 1

Number of PDI shares. Default is 1 (unprotected).

sdi: object

Secret Data Input port

bit_width: integer , { x | 8 x 32 }

Width of each share of SDI data (bits). Width of 'sdi_data' port is 'sdi.bit_width * sdi.shares'. Default is the same value as 'pdi.bit_width'

num_shares: integer , { x | 1 x 8 }

Number of SDI shares. Default is the same as 'pdi.num_shares'.

do: object

Data Output port. 'bit_width' is always the same as 'pdi.bit_width'

num_shares: integer , { x | 1 x 8 }

Number of DO output shares. Default is the same as 'pdi.num_shares'.

rdi: object

Random Data Input port.

bit_width: integer , { x | 0 x 2048 }
default: object
sca_protection: object [object Object]

Implemented countermeasures against side-channel attacks.

attacks: string[]

Family and type of side-channel analysis attack(s) this design is claims to be secure against. Examples: 'spa', 'dpa', 'cpa', 'timing', 'sifa', 'dfia', etc

string
dpa_order: integer , { x | 0 x 7 }

Claimed order of protectcion against differential power analysis. 0 means unprotected.
