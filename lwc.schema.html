Design

Type: object

Captures attributes of an LWC hardware implementations to enable automated benchmarking and validation.

 No Additional Properties

root name
Type: string

A unique identifier for the design. It can consist of English letters, digits, dashes, and underscores and must start with a letter.

Must match regular expression: ^[A-Za-z][-\w]+$

root description
Type: string

A short description of the design.

root author
Type: string or array of string

Author or list of authors and developers who have contributed to this implementation.

root url
Type: string

Uniform Resource Locator pointing to a webpage or source repository associated with this design or its author(s).

root license
Type: string or array of string

License or licenses covering this design's use and distribution. Use SPDX (ISO/IEC 5962:2021) short identifiers when applicable.
Example:

"SHL-2.1"

root version
Type: string

Design version.

Must match regular expression: ^(0|[1-9]\d*)(\.(0|[1-9]\d*)(\.(0|[1-9]\d*))?)?$
Example:

"0.0.1"

root rtl
Type: object

Details of the synthesizable RTL design

 No Additional Properties

root rtl sources
Type: array of string

Non-empty list of HDL source file paths in correct compilation order. All paths must be relative to the location of the configuration file. Path separator is / (slash) on all platforms. Paths are case-sensitive and should not contain whitespaces. HDL language is inferred from the file extension.

Each item of this array must be:

root rtl sources items
Type: string
Must match regular expression: ^[-\w\.]+(/[-\w\.]+)*$

root rtl includes
Type: array of string Default: []

Non-empty list of HDL include file paths, such as Verilog headers. Include files are not directly compiled but need to be present for elaboration of design. Order is arbitrary. All paths must be relative to the location of the configuration file. Path separator is / (slash) on all platforms. Paths are case-sensitive and should not contain whitespaces.

Each item of this array must be:

root rtl includes items
Type: string
Must match regular expression: ^[-\w\.]+(/[-\w\.]+)*$

root rtl top
Type: string Default: "LWC"

Name of top-level RTL entity/module.

root rtl clock
Type: object Default: {}

Top level RTL clock signal. Only a single clock is supported by LWC API

root rtl clock port
Type: string Default: "clk"

Name of the top-level RTL clock input.

root rtl reset
Type: object Default: {}

Top level reset signal. Only a single reset is supported by LWC API.

root rtl reset port
Type: string Default: "reset"

Name of top-level RTL reset input.

root rtl reset active_high
Type: boolean Default: true

Polarity of the reset signal. Active-high (positive) if true, otherwise active-low.

root rtl reset asynchronous
Type: boolean Default: false

Whether reset is asynchronous with respect to rtl.clock.

root rtl parameters
Type: object

Top-level design parameters or generics specified as a key-value map. The default value of each parameter is overridden by synthesis tool, simulator, testbench, or wrapper. For the best tool compatibility, we only support integer and string values.
Example:

{
    "G_NUM_SHARES": 2,
    "G_BACKDOOR": 0
}

All property whose name matches the following regular expression must respect the following conditions

Property name regular expression: .+
root rtl parameters .+
Type: integer or string

root rtl language
Type: object Default: {}

Information about Hardware Description/Design Language(s).

root rtl language vhdl
Type: object

Common VHDL features supported by all VHDL source files. VHDL files must have a .vhd or .vhdl extension.

root rtl language vhdl version
Type: enum (of string) Default: "1993"

VHDL language standard

Must be one of:

  • "1993"
  • "2000"
  • "2002"
  • "2008"

root rtl language vhdl synopsys
Type: boolean Default: false

Use of non-standard Synopsys packages which were placed in the IEEE namespace, e.g. 'stdlogicarith'. Dependence on such packages is strongly discouraged.

root rtl language verilog
Type: object

Common Verilog (pre-SystemVerilog) language features supported by all Verilog source files. Verilog files must have a .v extension.

root rtl language verilog version
Type: enum (of string) Default: "2001"

Verilog language standard

Must be one of:

  • "1995"
  • "2001"

root rtl language systemverilog
Type: object

SystemVerilog (IEEE 1800-2005 and onwards) language features supported by all SystemVerilog source files. SystemVerilog files must have a .sv extension.

root rtl language systemverilog version
Type: enum (of string) Default: "2009"

SystemVerilog language standard

Must be one of:

  • "2005"
  • "2009"

root lwc
Type: object

LWC-specific meta-data

root lwc algorithm
Type: string or array of string

LWC AEAD/Hash algorithm(s) supported by this design. Should follow SUPERCOP naming conventions and uniquely identify the scheme's variant and version. In case of duplicates, the second instance indicates support for AEAD and Hash algorithms with the same name.
Examples:

[
    "giftcofb128v1"
]

[
    "romulusn1v12"
]

[
    "gimli24v1",
    "gimli24v1"
]

root lwc input_sequence
Type: object Default: {}

Order in which different input segment types should be fed to PDI.

root lwc input_sequence encrypt
Type: array of enum (of string) Default: ["npub", "ad", "pt", "tag"]

Each item of this array must be:

root lwc input_sequence encrypt items
Type: enum (of string)

Must be one of:

  • "ad"
  • "pt"
  • "npub"
  • "tag"

root lwc input_sequence decrypt
Type: array of enum (of string) Default: ["npub", "ad", "ct", "tag"]

Each item of this array must be:

root lwc input_sequence decrypt items
Type: enum (of string)

Must be one of:

  • "ad"
  • "ct"
  • "npub"
  • "tag"

root lwc pt_block_bits
Type: integer

Algorithm's size of plaintext/ciphertext 'blocks' in bits. This is the number of bits that the algorithm operates upon during its basic operations. Potentially used for the evaluation of some performance metrics.

root lwc ad_block_bits
Type: integer

Algorithm's size of associated-data 'blocks' in bits. This is the number of bits that the algorithm operates upon during its basic operations. Potentially used for the evaluation of some performance metrics.

root lwc key_bits
Type: integer Default: 128

root lwc npub_bits
Type: integer Default: 128

root lwc tag_bits
Type: integer Default: 128

root lwc hash
Type: object

root lwc hash supported
Type: boolean Default: false

Whether this implementation supports hashing.

root lwc hash digest_bits
Type: integer Default: 128

Size of hash digest (output) in bits.

root lwc ports
Type: object Default: {}

Description of LWC ports.

root lwc ports pdi
Type: object

Public Data Input port

root lwc ports pdi bit_width
Type: integer Default: 32

Width of each share of PDI data (bits). Width of 'pdi_data' signal would be pdi.bit_width × pdi.num_shares bits.

Value must be greater or equal to 8 and lesser or equal to 32

root lwc ports pdi num_shares
Type: integer Default: 1

Number of PDI shares. Default is 1 (unprotected).

root lwc ports sdi
Type: object

Secret Data Input port

root lwc ports sdi bit_width
Type: integer Default: 32

Width of each share of SDI data (bits). Width of sdi_data signal would be sdi.bit_width × sdi.num_shares bits.

Value must be greater or equal to 8 and lesser or equal to 32

root lwc ports sdi num_shares
Type: integer Default: 1

Number of SDI shares. Default is the same as pdi.num_shares.

Value must be greater or equal to 1

root lwc ports rdi
Type: object

Random Data Input port.

root lwc ports rdi bit_width
Type: integer Default: 0

Value must be greater or equal to 0 and lesser or equal to 2048

root lwc sca_protection
Type: object

Implemented countermeasures against side-channel attacks.

 No Additional Properties

root lwc sca_protection target
Type: array of string

Type of side-channel analysis attack(s) against which this design is assumed to be secure.

Each item of this array must be:

root lwc sca_protection target items
Type: string

Examples:

[
    "spa",
    "dpa",
    "cpa",
    "timing"
]

[
    "dpa",
    "sifa",
    "dfia"
]

root lwc sca_protection masking_schemes
Type: array of string Default: []

Masking scheme(s) applied in this implementation. Could be name/abbreviation of established schemes (e.g., "DOM", "TI") or reference to a publication.

Each item of this array must be:

root lwc sca_protection masking_schemes items
Type: string

Examples:

[
    "TI"
]

[
    "DOM",
    "https://eprint.iacr.org/2022/000.pdf"
]

root lwc sca_protection order
Type: integer Default: 0

Claimed order of protectcion. 0 means unprotected.
